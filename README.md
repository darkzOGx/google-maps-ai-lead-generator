## ğŸ› Troubleshooting & FAQs

### â“ No results found / Very few results
**Solution:**
- Try broader search terms ("restaurants" instead of "italian restaurants with outdoor seating")
- Increase `maxResults` parameter to 500+
- Remove strict filters (set `minRating: 0`, `minReviews: 0`)
- Try different location formats ("San Francisco" vs "San Francisco, CA" vs "SF Bay Area")

### â“ No emails found (0% extraction)
**Check these settings:**
- âœ… Ensure `enrichment.extractEmails: true` is enabled
- âœ… Set `filters.hasWebsite: true` (can't extract emails without websites)
- âœ… Disable `fastMode` (fast mode skips detail pages where websites are found)
- âš ï¸ Some industries have low website adoption (e.g., restaurants, retail)

### â“ Actor runs out of memory or times out
**Solution:**
- Set `maxConcurrency: 2` or lower (default is 5, which may be too high)
- Reduce `maxResults` per query (try 250-500 instead of 1000+)
- Split large searches into multiple smaller runs
- Disable reviews extraction if not needed (`extractReviews: false`)

### â“ All leads scored as "F" grade
**Likely cause:** Your ICP (Ideal Customer Profile) doesn't match the scraped businesses
**Solution:**
- Check `scoring.idealCustomerProfile.industries` matches the categories you're scraping
- Adjust industry and location weights to better match your target market
- Use the score breakdown to understand why leads scored low

### â“ Proxy errors (SHADER, DATACENTER, RESIDENTIAL)
**Solution:**
- Set `proxy.apifyProxyGroups: []` (empty array) for auto-selection
- Don't specify proxy group names unless you know your plan has them
- Actor will automatically use the best available proxies from your Apify plan

### â“ Slow performance / Taking too long
**Solution:**
- Enable `fastMode: true` for 10x faster scraping (basic data only)
- Disable email extraction if not needed
- Use datacenter proxies instead of residential (faster, cheaper)
- Check you have enough compute units (actor pauses if account depleted)
